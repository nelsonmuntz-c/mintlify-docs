# Tendermint RPC

The Tendermint RPC provides direct access to the blockchain consensus layer.

## Endpoint

```
http://localhost:26657
ws://localhost:26657/websocket
```

## Transactions

### Broadcast Transaction (Sync)

Submit and wait for CheckTx:

```bash
curl -X POST 'http://localhost:26657/broadcast_tx_sync' \
  -d '{"tx": "base64-encoded-signed-tx"}'
```

Response:
```json
{
  "jsonrpc": "2.0",
  "id": -1,
  "result": {
    "code": 0,
    "data": "",
    "log": "",
    "hash": "ABC123..."
  }
}
```

### Broadcast Transaction (Async)

Submit without waiting:

```bash
curl -X POST 'http://localhost:26657/broadcast_tx_async' \
  -d '{"tx": "base64-encoded-signed-tx"}'
```

### Broadcast Transaction (Commit)

Submit and wait for block inclusion:

```bash
curl -X POST 'http://localhost:26657/broadcast_tx_commit' \
  -d '{"tx": "base64-encoded-signed-tx"}'
```

<Note>
`broadcast_tx_commit` waits for the transaction to be included in a block. Use `broadcast_tx_sync` for better performance in production.
</Note>

## ABCI Queries

### Direct Query

```bash

```

### Smart Contract Query

Query a CosmWasm contract:

```bash
# Base64 encode the query message
QUERY=$(echo -n '{"get_count":{}}' | base64)
CONTRACT="xyz1contract..."

curl "http://localhost:26657/abci_query?path=\"/cosmwasm.wasm.v1.Query/SmartContractState\"&data=..."
```

<Note>
For smart contract queries, the REST API is much simpler to use.
</Note>

## WebSocket Subscriptions

Connect to WebSocket for real-time events:

```
ws://localhost:26657/websocket
```

### Subscribe to New Blocks

```json
{
  "jsonrpc": "2.0",
  "method": "subscribe",
  "params": ["tm.event='NewBlock'"],
  "id": 1
}
```

### Subscribe to Transactions

```json
{
  "jsonrpc": "2.0",
  "method": "subscribe",
  "params": ["tm.event='Tx'"],
  "id": 2
}
```

### Subscribe to Specific Events

```json
{
  "jsonrpc": "2.0",
  "method": "subscribe",
  "params": ["tm.event='Tx' AND transfer.recipient='xyz1recipient...'"],
  "id": 3
}
```

### Unsubscribe

```json
{
  "jsonrpc": "2.0",
  "method": "unsubscribe",
  "params": ["tm.event='NewBlock'"],
  "id": 4
}
```

### JavaScript Example

```javascript
const ws = new WebSocket('ws://localhost:26657/websocket');

ws.onopen = () => {
  // Subscribe to new blocks
  ws.send(JSON.stringify({
    jsonrpc: '2.0',
    method: 'subscribe',
    params: ["tm.event='NewBlock'"],
    id: 1
  }));

  // Subscribe to contract events
  ws.send(JSON.stringify({
    jsonrpc: '2.0',
    method: 'subscribe',
    params: ["tm.event='Tx' AND wasm.contract_address='xyz1contract...'"],
    id: 2
  }));
};

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  if (data.result?.data?.type === 'tendermint/event/NewBlock') {
    console.log('New block:', data.result.data.value.block.header.height);
  } else if (data.result?.data?.type === 'tendermint/event/Tx') {
    console.log('Transaction:', data.result.data.value.TxResult);
  }
};
```

## Best Practices

1. **Use sync broadcast** - `broadcast_tx_sync` for production
2. **Poll for confirmation** - Don't rely on `broadcast_tx_commit`
3. **Handle WebSocket reconnects** - Connections can drop
4. **Paginate large queries** - Use `page` and `per_page` params
5. **Cache genesis** - It doesn't change
